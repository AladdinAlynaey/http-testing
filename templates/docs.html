Platform Overview

HTTP Playground is a production-grade educational platform for learning HTTP requests, REST APIs, CURL commands, authentication, security, and automation through hands-on practice with real API endpoints.

What You'll Learn

  • HTTP methods (GET, POST, PUT, DELETE, PATCH)
  • RESTful API design and best practices
  • CURL command-line usage
  • API authentication (JWT tokens & API keys)
  • File uploads via API
  • AI-powered endpoints (OpenRouter integration)
  • API security and rate limiting
  • n8n workflow automation

Difficulty Levels

  • Beginner ‚Äî No authentication needed. 15 scenarios using public endpoints.
  • Intermediate ‚Äî Requires API key. 10 scenarios with CRUD operations.
  • Advanced ‚Äî Requires login. 5 AI-powered scenarios.

Quick Start

Open your terminal and try these commands immediately ‚Äî no setup required:

# Get all books curl https://n8nhttp.alaadin-alynaey.site/api/books
# Check the weather in Tokyo curl https://n8nhttp.alaadin-alynaey.site/api/weather?city=tokyo
# Create a new task curl -X POST https://n8nhttp.alaadin-alynaey.site/api/tasks \ -H "Content-Type: application/json" \ -d '{"title":"Learn REST APIs","priority":"high"}'

Base URL

https://n8nhttp.alaadin-alynaey.site

All API endpoints are prefixed with /api/. For example: /api/books

Authentication Guide

Step 1: Register an Account

curl -X POST https://n8nhttp.alaadin-alynaey.site/api/auth/register \ -H "Content-Type: application/json" \ -d '{"username":"myuser","email":"me@example.com","password":"mypassword123"}'

Step 2: Wait for Admin Approval

Your account starts in pending status. An admin must approve it before you can login.

Step 3: Login

curl -X POST https://n8nhttp.alaadin-alynaey.site/api/auth/login \ -H "Content-Type: application/json" \ -d '{"username":"myuser","password":"mypassword123"}'

Response includes access_token, refresh_token, and api_key.

Step 4: Use Your Token

# Using JWT token curl -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \ https://n8nhttp.alaadin-alynaey.site/api/auth/me # Using API key curl -H "X-API-Key: YOUR_API_KEY" \ https://n8nhttp.alaadin-alynaey.site/api/students

API Keys

API keys are automatically generated when your account is approved. Pass your key using:

  • Header: X-API-Key: nhk_your_key_here
  • Query parameter: ?api_key=nhk_your_key_here

API keys grant access to intermediate endpoints (Students, Files, Blog write, Inventory).

Error Codes

Code Meaning Common Cause
400 Bad Request Missing required fields or invalid data
401 Unauthorized Missing or invalid token/API key
403 Forbidden Account not approved or insufficient role
404 Not Found Resource doesn't exist
409 Conflict Duplicate username/email
413 Too Large File exceeds 5MB limit
429 Rate Limited Too many requests
500 Server Error Internal server issue
503 Unavailable AI service not configured

Rate Limits

Endpoint Type Limit Window
General API 200 requests Per minute
Login 10 attempts Per minute
Register 5 attempts Per minute
AI Endpoints 10 requests Per minute

Exceeding limits returns 429 Too Many Requests. Failed login attempts trigger account lockout after 5 failures (15 min cooldown).

Security

Security Headers

  • Content-Security-Policy (CSP) ‚Äî Restricts resource loading
  • Strict-Transport-Security (HSTS) ‚Äî Forces HTTPS
  • X-Frame-Options: DENY ‚Äî Prevents clickjacking
  • X-Content-Type-Options: nosniff ‚Äî Prevents MIME sniffing
  • X-XSS-Protection ‚Äî XSS attack mitigation
  • Referrer-Policy ‚Äî Controls referrer information

Protection Measures

  • JWT-based sessions with expiration & refresh flow
  • API key authentication with scoping
  • Rate limiting per IP and endpoint
  • Brute-force protection with account lockout
  • Request fingerprinting
  • Input sanitization with Bleach
  • Secure file uploads (type/size validation)
  • Full audit logging of all sensitive actions
  • CORS with strict origin whitelisting

n8n Integration

Use this API with n8n for workflow automation.

Example: Auto-create tasks from Telegram

  1. Set up an n8n Telegram trigger node
  2. Add an HTTP Request node:
    • Method: POST
    • URL: https://n8nhttp.alaadin-alynaey.site/api/tasks
    • Headers: Content-Type: application/json
    • Body: {"title": "{{$json.message.text}}", "priority": "medium"}

Example: Monitor inventory and send alerts

  1. Schedule trigger (every hour)
  2. HTTP GET: https://n8nhttp.alaadin-alynaey.site/api/inventory?low_stock=true
  3. IF node: Check if count > 0
  4. Send email/Slack notification with low stock items

Example: Blog to social media

  1. Webhook trigger
  2. Create blog post via API
  3. Share to Twitter/Telegram using n8n social nodes

AI Usage Guide

AI endpoints use OpenRouter for language model access.

Available AI Features

  • Generate ‚Äî Create text from a prompt (max 2000 chars)
  • Summarize ‚Äî Condense long text (20-5000 chars)
  • Classify ‚Äî Categorize text (custom or default categories)
  • Validate ‚Äî Check grammar, code, email, or API responses
  • Chat ‚Äî Interactive learning assistant with context modes

Chat Context Modes

  • general ‚Äî General teaching assistant
  • api ‚Äî REST API and HTTP expert
  • security ‚Äî Cybersecurity expert
  • python ‚Äî Python programming tutor

Safety

  • All prompts are sanitized before processing
  • Output is limited to prevent abuse
  • 10 requests/minute rate limit
  • All AI interactions are logged for auditing
